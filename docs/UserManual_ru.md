## 1. Что это такое

VibroVarioAuto — это барометрический вариометр для параплана, дельтаплана, планёра.
Собран на базе умных часов Watchy (ESP32 + E-Ink экран).

### Чем он отличается от других вариометров

- **Беззвучный режим** — вибромотор на запястье. Никаких писков в ухо.
- **Звуковой режим** — можно подключить буззер. Тональность как у Brauneiger.
- **E-Ink экран** — читается на ярком солнце, не бликует. Не расходует энергию в покое.
- **Eнергосбережение** — в режиме часов работает неделями, в полёте 10+ часов.
- **Умный сон** — часы сами просыпаются каждую минуту, проверяют, не взлетели ли вы.
- **Запись трека** — каждый полёт записывается во flash. Потом — выгрузка по WiFi.
- **Вес** — всего 24 грамма с батареей.

## 2. Как прибор устроен внутри

В часах установлены:

| Датчик | Что измеряет | Зачем |
|--------|-------------|-------|
| BMP390 | Давление воздуха | Вычисляет высоту и вертикальную скорость |
| BMA423 | Ускорение (3 оси) | Помогает фильтру реагировать быстрее |
| PCF8563 | Точное время | Часы реального времени, не сбрасываются |
| Вибромотор | — | Тактильная обратная связь на запястье |

### Как работают датчики вместе

**Барометр (BMP390)** измеряет давление с частотой 50 раз в секунду.
По давлению вычисляется высота над уровнем моря по стандартной атмосфере.

**Акселерометр (BMA423)** — это датчик, который чувствует движение
и наклон. Он используется для двух вещей:

1. **Определение вертикали** — где верх, где низ. Часы могут быть
   повёрнуты как угодно (на запястье, в кармане, на подвеске),
   акселерометр всегда находит направление «вниз» по силе тяжести.

2. **Быстрый отклик** — когда вы входите в термик, тело чувствует
   ускорение вверх раньше, чем барометр заметит изменение давления.
   Акселерометр даёт сигнал мгновенно, а барометр — точное значение
   без дрейфа. Они объединяются в **комплементарном фильтре**.

**Итог**: варио работает быстро (как с акселерометром) и точно (как с барометром),
и неважно, как повёрнуты часы.

## 3. Как выглядит экран

### Экран часов (режим CLOCK)

Когда прибор не в полёте, он показывает часы:

```
┌──────────────────────┐
│  3.95V   87%         │    ← напряжение батареи и заряд в %
│                      │
│       14:32          │    ← текущее время (крупно)
│                      │
│      01.05           │    ← дата (число.месяц)
│                      │
└──────────────────────┘
```

В этом режиме часы большую часть времени спят.
Просыпаются 1 раз в минуту — обновить время на экране и проверить
давление (не взлетели ли вы). После обновления — снова спят.

### Экран полёта (режим RUNNING)

После старта полёта экран меняется:

```
┌──────────────────────┐
│ 00:12:34  24.5c  87% │    ← время полёта | °C | заряд
│                      │
│ Start, m    Sea, m   │    ← заголовки
│    +245       589    │    ← подъём от старта | высота над морем
│                      │
│ Vario  max+3.2 min-1.1│   ← статистика за полёт
│        +1.7          │    ← ВЕРТИКАЛЬНАЯ СКОРОСТЬ (крупно)
└──────────────────────┘
```

**Верхняя строка:**
- Слева — сколько времени вы уже в полёте
- В центре — температура воздуха
- Справа — заряд батареи в процентах

**Средняя часть:**
- `Start, m` — на сколько метров вы поднялись относительно старта
- `Sea, m` — абсолютная высота над уровнем моря

**Нижняя часть:**
- `max+X.X min-Y.Y` — рекорды за полёт
- Крупное число — текущая вертикальная скорость (±м/с)

## 4. Режимы работы

У прибора 6 режимов:

```
DEEP SLEEP  ──►  CLOCK  ──►  SETTINGS  ──►  CALIBRATING  ──►  RUNNING  ──►  STOPPED
   (спит)        (часы)    (настройки)     (калибровка)      (полёт)      (пауза)
                     ▲                                                 │
                     └─────────────────────────────────────────────────┘

                     CLOCK  ──►  WEB_EXPORT
                    (часы)      (WiFi AP 15 мин)
```

### DEEP SLEEP — глубокий сон

Потребление: **~5 мкА** (батарея не садится месяцами).

- Экран пустой, все датчики выключены
- Проснуться можно **любой кнопкой** (UP, OK, DOWN)
- При пробуждении — самодиагностика (SELF-TEST)

### CLOCK — режим часов

Потребление: **~30 мкА** в среднем.

- На экране время, дата, заряд батареи
- Просыпается раз в 60 сек — проверить давление (автостарт полёта)
- Кнопки: DOWN → настройки, OK → старт полёта, UP → сон

### WEB_EXPORT — WiFi-экспорт

- Запускается из Settings: строка `WiFi: [OFF]` → OK
- Создаёт точку доступа `VibroVario`
- В браузере на телефоне/ноутбуке открыть `http://192.168.4.1/`
- Можно скачать CSV трека или настроить часы/высоту
- Автоотключение через 15 минут. UP — выйти вручную.

### RUNNING — режим полёта

Потребление: **~7 мА**.

- Всё включено: барометр 50 Гц, акселерометр, дисплей, звук и вибрация
- Вертикальная скорость обновляется мгновенно
- Запись трека: время + высота, 1 раз/сек
- Экран обновляется 1 раз в секунду
- **Автопосадка**: через 5 мин без движения → CLOCK

### STOPPED — полёт на паузе

- Все показания заморожены
- UP → CLOCK, DOWN → новый полёт

## 5. Кнопки — что и когда нажимать

На корпусе 4 кнопки. Используются 3.

**Важно:** из глубокого сна устройство пробуждает любая кнопка — UP, OK или DOWN.
При пробуждении выполняется самодиагностика.

```
        ┌──────────────────────┐
        │                      │
   (26) │                      │ (25) UP
  SETUP │        ЭКРАН         │  ← Сон / Выйти
        │                      │
        │                      │
        └──────────┬───────────┘
                   │
              (35) │ (4) OK
              DOWN │  Старт/Стоп/Ввод
              Настр│
```

### Кнопка UP (GPIO 25, верхняя правая)
| Режим | Действие |
|-------|----------|
| CLOCK | Уйти в сон (DEEP SLEEP) |
| RUNNING | Стоп полёта → CLOCK |
| STOPPED | CLOCK |
| SETTINGS | CLOCK |
| WEB_EXPORT | Выключить WiFi → CLOCK |

### Кнопка OK (GPIO 4, нижняя левая)
| Режим | Действие |
|-------|----------|
| CLOCK | Старт полёта |
| RUNNING | Стоп полёта (пауза, статистика) |
| STOPPED | Сброс RTC на 00:00 |
| SETTINGS | Toggle / войти в редактирование |

### Кнопка DOWN (GPIO 35, нижняя правая)
| Режим | Действие |
|-------|----------|
| CLOCK | Настройки (Settings) |
| STOPPED | Новый полёт |
| SETTINGS | Следующая строка |

## 6. Как пользоваться в полёте — пошагово

### Перед вылетом

1. Проверьте заряд батареи — > 50%.
2. Закрепите часы на запястье или подвеске.
3. Убедитесь, что вибро чувствуется.
4. **Проверьте настройки** (DOWN на часах):
   - Buzzer / Vibro
   - Time — точное время
   - Alt — высота вашего места над уровнем моря (для авторасчёта QNH)

### Взлёт

**Ручной:** OK → "Calibrating..." (2 сек, не двигать) → полёт.

**Автоматический:** часы заметят набор высоты → "FLIGHT! Press DOWN" → OK/DOWN.

### В полёте

- **Буззер**: частое пищание = подъём, редкое гудение = снижение, непрерывный тон = авария
- **Вибро**: 1-4 импульса в зависимости от силы подъёма
- **Экран**: высота, скорость, время полёта
- **Запись трека**: идёт автоматически, 1 раз/сек

### Посадка

- **Ручной**: OK (стоп) → UP (часы)
- **Авто**: через 5 мин без движения сам остановит

### После полёта — выгрузка трека

1. Settings (DOWN) → WiFi → OK
2. Телефон → WiFi → подключиться к `VibroVario`
3. Браузер → `http://192.168.4.1/`
4. Таблица полётов → скачать CSV нужного полёта или все
5. UP на часах — выключить WiFi

## 7. Экран FLIGHT

В режиме CLOCK часы просыпаются каждые 60 секунд и делают 1 замер давления.
Если с прошлой проверки высота выросла больше чем на 0.75 м/с в среднем,
часы показывают "FLIGHT! Press DOWN". Нажмите DOWN или OK для старта.

## 8. Экран настроек (SETTINGS)

При нажатии DOWN на экране часов открывается меню настроек:

| Строка | Параметр | Действие OK |
|--------|----------|-------------|
| 0 | Buzzer: [ON/OFF] | Toggle |
| 1 | Vibro: [ON/OFF] | Toggle |
| 2 | Time: HH:MM | Войти в редактирование (UP/DOWN менять, OK следующий разряд) |
| 3 | Alt: 300m | Ввести высоту места (UP/DOWN +5/-5, OK сохранить + QNH) |
| 4 | WiFi: [OFF] | Запустить WiFi-экспорт |
| 5 | Sens: [4] | Ввести чувствительность (UP/DOWN 0..9, OK сохранить) |

Настройки сохраняются в RTC-памяти и действуют до следующего изменения.

### Чувствительность (Sens: 0-9)

Регулирует скорость реакции вариометра на вертикальные движения.

| Значение | Реакция | Для кого |
|----------|---------|----------|
| 0-2 | **Быстро** — реагирует мгновенно, больше шума | Акро, спидфлай, быстрый поиск термиков |
| 3-5 | **Сбалансировано** (по умолч. 4) — чисто, отзывчиво | Обычный параплан, маршрут |
| 6-9 | **Медленно** — гладко, игнорирует мелкие тычки | Парение, слабые потоки, чувствительные пилоты |

Внутри меняет частоту среза комплементарного фильтра: меньше — акселерометр доминирует (быстрее), больше — барометр доминирует (плавнее).

### Настройки через браузер

В режиме WiFi-экспорта откройте `http://192.168.4.1/settings`:
- Часы: Hour, Minute
- Высота места: Alt (m)
- Save → запись в RTC и пересчёт QNH

## 9. Самодиагностика (SELF-TEST)

При каждом пробуждении по кнопке выполняется автоматическая проверка:

| Что проверяется | При ошибке |
|----------------|-----------|
| Кнопки DOWN, OK | `BTN: DOWN OK` |
| BMP390 | `BARO FAIL` |
| BMA423 | `ACCEL FAIL` |
| Батарея (< 3.4V) | `LOW BATT` |
| BMP в полёте | `SENSOR FAIL` на экране |

При ошибке — экран SELF-TEST на 5 секунд, затем часы.

## 10. Про питание

| Режим | Потребление | Время работы |
|-------|-------------|--------------|
| DEEP SLEEP | 5 мкА | Месяцы |
| CLOCK | 30 мкА | Недели |
| RUNNING | 7 мА | 10+ ч |
| RUNNING + буззер | ~10 мА | 8+ ч |
| WEB_EXPORT | ~80 мА | ~15 мин (ограничено таймером) |

Батарея 200 мАч, 4.2В = 100%, 3.3В = 0%.

## 11. Запись трека

- **Частота**: 1 запись/сек в полёте
- **Формат**: flightNum + время + высота (8 байт/точка)
- **Ёмкость**: ~18 полётов по 3 часа, затем кольцевая перезапись
- **Выгрузка**: WiFi → браузер → CSV

Каждая запись содержит: номер полёта, время (HH:MM:SS), высоту MSL (метры).

## 12. Неисправности и частые вопросы

**Q: Экран пустой, ничего не происходит.**
A: Нажмите любую кнопку (UP/OK/DOWN). Если не реагирует — подключите USB.

**Q: Прибор завис.**
A: Сторожевой таймер перезагрузит его через 10 секунд. Если нет — подключите USB.

**Q: Варио показывает ерунду.**
A: UP → CLOCK → OK → калибровка (2 сек неподвижно).

**Q: На экране SENSOR FAIL.**
A: Отказ BMP390. Проверьте контакты. Если временный — перезагрузится.

**Q: Как выгрузить трек?**
A: Settings → WiFi → OK → подключиться к VibroVario → браузер 192.168.4.1/

**Q: Батарея быстро садится в режиме WiFi.**
A: Так и задумано — WiFi жрёт ~80 мА. Автоотключение через 15 минут.

## 13. Установка и компиляция (для сборки своими руками)

### Структура проекта

```
VibroVario/                    ← папка скетча (при клоне через git)
├── VibroVario-main.ino        ← главный файл (открывать в Arduino IDE)
├── config.h                   ← пины, константы, настройки фильтра
├── VarioEMA.h                 ← класс комплементарного фильтра
├── README.md                  ← описание проекта (русский)
└── docs/
    ├── UserManual_ru.md       ← эта инструкция
    └── UserManual_en.md       ← English manual
```

### Что нужно для сборки

1. **Arduino IDE** (скачать с arduino.cc) — версия 2.x или 1.8+
2. **Плата ESP32** — в меню Файл → Настройки → «Дополнительные ссылки для менеджера плат» вставить:
   `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`
3. **Установить ESP32** — Инструменты → Плата → Менеджер плат → найти "ESP32" → установить
4. **Библиотеки** — Менеджер библиотек (Скетч → Подключить библиотеку → Управлять библиотеками):
   - `GxEPD2` (последняя версия)
   - `Adafruit BMP3XX`
   - `Adafruit Unified Sensor`
5. **Подключить Watchy** через USB
6. **Выбрать плату** — Инструменты → Плата → ESP32 Arduino → ESP32 Dev Module
7. **Параметры платы:**
   - Flash Size: `4MB (32Mb)`
   - Partition Scheme: `Default 4MB with spiffs (1.2MB APP/1.5MB SPIFFS)`
8. **Открыть файл** — Файл → Открыть → выбрать папку `VibroVario-main` → откроется `VibroVario-main.ino`
9. **Нажать Upload** (→)

Файлы `config.h` и `VarioEMA.h` подхватятся автоматически — их не нужно добавлять вручную.

### Компиляция через CLI (для продвинутых)

```bash
arduino-cli core install esp32:esp32
arduino-cli lib install "GxEPD2"
arduino-cli lib install "Adafruit BMP3XX"
arduino-cli compile --fqbn esp32:esp32:esp32 .
```
